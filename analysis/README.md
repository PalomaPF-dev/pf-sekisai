# 積載計画・不足分析 — 入力データ仕様と加工ルール

清洲組立B棟で使用する組立部品(点滅基装組立ほか5品目)について、
在庫・使用予定・入荷予定・配船情報から不足を判定し、積載計画の
インプットを作るための整理。基準日は 2026-07-16 抽出のデータ。

## 入力ファイル(5種)

| # | 帳票名 | 役割 | 主なキー・数量列 |
|---|--------|------|------------------|
| ① | 現状在庫一覧照会 | その時点の在庫 | 品目ＣＤ / 場所ＣＤ / 在庫数量 |
| ② | 受払予定一覧照会 | 未確定の使用予定(投入計画・オーダ保存) | 品目ＣＤ / 予定日付 / 基準単位出庫数量 |
| ③ | 投入一覧照会 | 確定済の使用予定(指図済・着手中) | 子品目ＣＤ / 投入予定日 / 投入残数量 |
| ④ | 入荷実績一覧登録 | 出荷実績と配船番号のセット(未入荷=輸送中〜入荷予定) | 品目ＣＤ / 入荷予定日 / 入荷予定数量 / 入荷連絡備考(=配船番号) |
| ⑤ | 配船マスタ | 配船情報(週コード・船名・出港/入港日等) | wk_cd / shp_air_typ / shp_air_no |

## ②と③の結合(「泣き別れ」の解消)

使用予定は指図確定のタイミングで②から③へ移るため、単独ではどちらも
全体を表さない。実使用予定は次のルールで結合する。

- **③(確定分)**: 数量は「投入残数量」(投入済み分を除いた残り)。
  投入予定日が基準日より過去の行は未消化の残なので**基準日扱いに繰上げ**る。
  (20260716データでは 3/31〜7/17 に分布、残合計 9,312)
- **②(未確定分)**: 数量は「基準単位出庫数量」。予定日付は基準日翌日以降
  (7/17〜12/18)に分布。
- **境界日(7/17)は両方に行が存在する**(確定済み分＋未確定分)ため、
  重複ではなく**単純合算**する。
- 実使用予定 = ③投入残(日別) + ②出庫予定(日別)

## ④と⑤の紐付け(配船番号)

- ④の配船番号は**「入荷連絡備考」列**に入る。形式は `Y{週}-{区分}{記号}`
  (例: `Y288-1K`、区分1=船便)。航空便は `YA{週}-{連番}` (例: `YA289-7`)。
- ⑤配船マスタの配船番号は `{週}-{記号}` (船便) / `{週}-{連番}` (航空便)。
- **注意**: 今回の⑤抽出は週220〜283(出港日 2025-06〜2026-08-29)のみで、
  ④が参照する週285〜289のレコードが未登録。船名・入港日との突合には
  最新の配船マスタ抽出が必要。

## 不足判定のロジック

```
予定在庫(日) = 前日残 + ④入荷予定(日) - 実使用予定(日)
初期値      = ①在庫数量(清洲組立B棟 Z0E2000)
不足        = 予定在庫 < 0 となる最初の日
```

- ①の引当可能数量は全行0のため在庫数量ベースで算定。
- ④の入荷予定は 8/17 まで。**8/18以降の需要はすべて今後の配船
  (週290〜相当)への積載計画の対象**になる。

## 積載計画案のロジック

週次バケットで「不足させないために各週に何個入荷(=積載)させるべきか」を算出する。

```
必要追加入荷(週) = MAX(0, 当週需要 + 安全在庫係数×翌週需要 - 前週末在庫 - 登録済入荷(週))
週末在庫(週)     = 前週末在庫 + 登録済入荷 + 必要追加入荷 - 当週需要
```

- **安全在庫係数**(既定 0.5 = 翌週需要の半分)と**海上リードタイム**(既定 3週)は
  Excel上の黄色セルで変更可能。変更すると計画全体が再計算される。
- 各入荷週に対し **積込目安週 = 入荷週 - リードタイム** を表示。
- 配船番号は登録済み週は④の実配船、以降は最終配船(Y289)から週次で外挿した
  推定番号(Y290〜)を表示。
- **登録済入荷期間内(〜8/17、グレー網掛け週)に必要追加入荷が出る品目は
  船便では間に合わない**ため、航空便・入荷前倒しの検討対象
  (20260716データでは ﾊｲｾﾝｿｳｸﾐﾀﾃR が該当)。

## ⑤配船マスタとの突合

`parse_vessel_ref()` が④の配船番号をマスタキーに変換して突合する
(`Y288-1K` → `288-K`、`YA289-7` → `289-7`)。配船別入荷予定シートに
船名・コンテナ・出港日を表示する。今回の⑤抽出には週285〜289が無いため
全て空欄になっており、最新マスタで再実行すれば自動的に埋まる。

## スクリプト

`build_shortage_analysis.py` が5ファイルを読み、上記ルールで
`積載計画_不足分析_YYYYMMDD.xlsx` (サマリ / 実使用予定 / 在庫推移 /
配船別入荷予定 / 積載計画案 の5シート)を生成する。

```
python3 build_shortage_analysis.py --input-dir <5ファイルのあるフォルダ> --out 出力.xlsx
```

ファイル名は問わず、ヘッダ行の内容で①〜⑤を自動判別する。
